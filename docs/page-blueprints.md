# Page Blueprints

## Purpose

Define allowed future page families after Phase 0. These are structures and controls, not publish-ready content.

## Approved Candidate Families

### Comparison Pages

Use for controlled `X vs Y` or `alternative to X` style pages only when:

- the comparison is factual
- the differentiators are supportable
- legal and brand review is satisfied

### Borrower Profile Pages

Use for audience-specific guidance pages when the target borrower segment is clearly defined and non-deceptive.

### Industry Pages

Use for industry-specific financing needs where the page can carry a meaningful operational angle, not keyword stuffing.

### Tool Pages

Use for utility pages such as calculators, checklists, or evaluation helpers only if they provide genuine user value.

## Required Inputs Per Blueprint

- page purpose
- search intent
- target user
- canonical rule
- disclosure requirement
- schema requirement
- internal linking plan
- QA checklist

## Disallowed Behaviors

- mass-producing pages with thin copy
- publishing without a verified canonical strategy
- inventing borrower outcomes or customer proof
- mixing brand names inconsistently
- shipping schema that does not match visible page intent

## Approval Gate

No blueprint should move into generation until:

- Phase 0 done criteria are met
- keyword cluster data is populated
- schema starter exists
- verification prompt is ready
- operator sign-off is captured
